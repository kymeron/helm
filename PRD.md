# HELM 产品需求文档 (PRD)

> **版本**: v0.1
> **日期**: 2026-06-19
> **状态**: 草案

---

## 一、产品概述

### 1.1 产品名称
**HELM** —— 取自轮船的"舵"（helm），寓意指引方向、掌控航向。

### 1.2 产品定位
一款面向个人使用的 PC 端驾驶舱看板应用，用于记录想法、疑问与探索，跟踪任务进度，并通过统计图表辅助自我管理与决策。

### 1.3 目标用户
- 下班后有时间、有想法，但缺乏清晰执行路径的个人用户
- 希望对个人"想法池"进行结构化管理与跟踪的人

### 1.4 核心价值
- **记录**: 快速捕捉灵感、疑问、探索方向，避免遗忘
- **跟踪**: 以看板形式可视化任务状态与进度
- **分析**: 通过统计指标与图表，洞察自己的投入分布与完成情况
- **指引**: 如舵一般，帮助用户明确"下一步做什么"

## 二、需求背景

### 2.1 用户痛点
| 痛点 | 描述 |
|------|------|
| 想法散落 | 脑海中有很多想做的事，但没有统一记录，容易遗忘 |
| 不知从何开始 | 事情多但无优先级，下班后陷入"选择困难" |
| 进度不可见 | 正在做的事缺乏跟踪，容易半途而废 |
| 缺乏复盘 | 没有数据支撑，无法了解自己的投入与产出 |

### 2.2 解决方案
构建一个集"记录 - 看板 - 统计"于一体的个人驾驶舱：
- 用卡片化看板承载任务，状态一目了然
- 用统计图表量化投入与完成情况
- 用简约极客的界面降低使用负担，提升专注度

## 四、功能需求

### 4.1 功能架构

```
HELM
├── 任务管理
│   ├── 任务卡片（创建/编辑/删除）
│   ├── 任务类型（Idea / Issue / Exploration）
│   └── 任务状态（待办 / 进行中 / 已完成）
├── 看板视图
│   ├── 三栏看板（按状态分列）
│   ├── 卡片拖拽（状态流转）
│   └── 筛选与排序
├── 统计分析
│   ├── 核心指标卡片
│   ├── 图表可视化
│   └── 趋势分析
└── 设置
    └── 偏好配置
```

### 4.2 任务管理

#### 4.2.1 任务卡片字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识 |
| 标题 (title) | string | 是 | 简短描述，≤50字 |
| 描述 (description) | text | 否 | 详细说明，支持 Markdown |
| 类型 (type) | enum | 是 | `idea` / `issue` / `exploration` |
| 状态 (status) | enum | 是 | `todo` / `in_progress` / `done` |
| 优先级 (priority) | enum | 否 | `low` / `medium` / `high`，默认 `medium` |
| 标签 (tags) | string[] | 否 | 自定义标签，便于分类 |
| 创建时间 (createdAt) | datetime | 是 | 自动生成 |
| 更新时间 (updatedAt) | datetime | 是 | 自动更新 |
| 完成时间 (completedAt) | datetime | 否 | 状态变更为 done 时记录 |

#### 4.2.2 任务类型说明

| 类型 | 标识 | 含义 | 示例 |
|------|------|------|------|
| Idea | 💡 idea | 想法、灵感 | "学一门乐器" |
| Issue | ❓ issue | 疑问、待解决的问题 | "是否值得深入学习 Rust？" |
| Exploration | 🧭 exploration | 探索、调研类任务 | "调研 2026 年主流 AI Agent 框架" |

#### 4.2.3 任务状态流转

```
┌──────────┐  开始  ┌──────────────┐  完成  ┌──────────┐
│  待办     │ ────> │   进行中      │ ────> │  已完成    │
│  todo     │       │  in_progress │       │   done   │
└──────────┘       └──────────────┘       └──────────┘
     ↑                    │  暂停               │
     └────────────────────┘                     │
     ↑                                         │
     └───────────── 重新打开 ────────────────────┘
```

- 允许状态回退（如进行中→待办、已完成→进行中）
- 状态变更时自动记录 `updatedAt`
- 变更为 `done` 时自动记录 `completedAt`

#### 4.2.4 卡片操作

| 操作 | 说明 |
|------|------|
| 新建 | 点击"新建"按钮，弹出表单填写任务信息 |
| 编辑 | 点击卡片进入编辑模式，修改字段 |
| 删除 | 卡片支持删除，需二次确认 |
| 状态变更 | 拖拽卡片至目标列，或卡片内切换状态 |
| 归档 | 已完成任务可归档，从看板移除但保留数据 |

### 4.3 看板视图

#### 4.3.1 布局
- 三列布局，对应三种状态：**待办** | **进行中** | **已完成**
- 每列顶部显示列标题与任务计数
- 卡片以列表形式纵向排列于各列中

#### 4.3.2 卡片展示信息
- 标题
- 类型标识（图标 + 颜色）
- 优先级标识
- 标签
- 创建/完成时间（相对时间，如"3天前"）

#### 4.3.3 筛选与排序
| 维度 | 选项 |
|------|------|
| 按类型筛选 | 全部 / idea / issue / exploration |
| 按优先级筛选 | 全部 / high / medium / low |
| 按标签筛选 | 全部 / 指定标签 |
| 排序 | 创建时间 / 优先级 / 更新时间 |

#### 4.3.4 拖拽交互
- 支持鼠标拖拽卡片在列间移动
- 拖拽时卡片半透明，目标列高亮
- 释放后状态自动更新

### 4.4 统计分析

#### 4.4.1 核心指标卡片

页面顶部展示关键指标卡片：

| 指标 | 说明 |
|------|------|
| 任务总数 | 全部任务数量 |
| 待办数 | 状态为 todo 的任务数 |
| 进行中数 | 状态为 in_progress 的任务数 |
| 已完成数 | 状态为 done 的任务数 |
| 完成率 | 已完成 / 总任务 |
| 本周新增 | 近 7 天新建任务数 |
| 本周完成 | 近 7 天完成任务数 |

#### 4.4.2 图表

| 图表 | 类型 | 说明 |
|------|------|------|
| 任务类型分布 | 环形图 | idea / issue / exploration 占比 |
| 状态分布 | 横向条形图 | 三种状态数量对比 |
| 完成趋势 | 折线图 | 近 30 天每日完成数量趋势 |
| 类型×状态矩阵 | 堆叠柱状图 | 各类型在不同状态下的分布 |
| 标签热度 | 词云 / 条形图 | 出现频次最高的标签 Top N |

#### 4.4.3 时间范围
- 统计支持时间范围切换：今日 / 本周 / 本月 / 全部

## 五、页面设计

### 5.1 设计风格
- **简约**: 去除冗余装饰，信息密度适中，留白合理
- **极客**: 深色主题为主，等宽字体点缀，数据驱动感
- **高效**: 关键操作触手可及，减少点击层级

### 5.2 视觉规范

| 规范项 | 说明 |
|--------|------|
| 主题 | 深色主题（dark mode）为主 |
| 主色调 | 冷色调，如深蓝/青色系，呼应"航海/舵"意象 |
| 强调色 | 用于高亮与交互反馈 |
| 字体 | 系统无衬线字体 + 等宽字体（用于数据/代码） |
| 类型色板 | idea=琥珀色 / issue=红色 / exploration=青色 |

### 5.3 页面布局

### 5.4 交互细节
- 卡片悬停时轻微上浮 + 阴影
- 拖拽时目标列高亮边框
- 指标卡片悬停展示详细 tooltip
- 图表支持 hover 显示数据点详情
- 新建/编辑任务使用模态框（Modal），不跳转页面

## 六、数据模型

### 6.1 任务（Task）

```typescript
interface Task {
  id: string;
  title: string;
  description?: string;
  type: 'idea' | 'issue' | 'exploration';
  status: 'todo' | 'in_progress' | 'done';
  priority: 'low' | 'medium' | 'high';
  tags: string[];
  createdAt: string;   // ISO 8601
  updatedAt: string;   // ISO 8601
  completedAt?: string; // ISO 8601
}
```

### 6.2 存储
- 本期采用浏览器本地存储（localStorage / IndexedDB）
- 数据结构为 Task 数组
- 后续可扩展为后端 API + 数据库
