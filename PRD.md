> **版本**: v0.3
> **日期**: 2026-06-20
> **状态**: 进行中(与代码同步)

---

## 一、产品概述

### 1.1 产品名称

**HELM** —— 取自轮船的「舵」(helm),寓意指引方向、掌控航向。

### 1.2 产品定位

一款面向个人使用的驾驶舱看板 Web 应用。桌面端为信息密集型的「驾驶舱」视图,平板与手机端做对应响应式压缩,但保留全部核心能力。支持**本地优先**使用,可选启用**局域网同步**实现多设备互通。

### 1.3 目标用户

- 下班后有时间、有想法但缺乏清晰执行路径的个人用户
- 希望对个人「想法池」进行结构化管理与跟踪的人
- 在多个设备(PC + iPad + 手机)间切换、需要在同 WiFi 下保持数据一致的个人用户

### 1.4 核心价值

- **记录**: 快速捕捉灵感、疑问、探索方向,避免遗忘
- **跟踪**: 以看板形式可视化任务状态与进度
- **分析**: 通过统计指标与图表(含活动热力图),洞察投入分布与完成情况
- **指引**: 如舵一般,帮助用户明确「下一步做什么」

## 二、需求背景

### 2.1 用户痛点

| 痛点 | 描述 |
|------|------|
| 想法散落 | 脑海中有很多想做的事,但没有统一记录,容易遗忘 |
| 不知从何开始 | 事情多但无优先级,下班后陷入「选择困难」 |
| 进度不可见 | 正在做的事缺乏跟踪,容易半途而废 |
| 缺乏复盘 | 没有数据支撑,无法了解自己的投入与产出 |
| 多设备割裂 | 在 PC 记的想法,回家想从 iPad / 手机查看却找不到 |

### 2.2 解决方案

构建一个集「记录 - 看板 - 统计 - 多设备」于一体的个人驾驶舱:

- 用卡片化看板承载任务,状态一目了然
- 用统计图表(含 GitHub 风格活动热力图)量化投入与完成情况
- 用简约极客的深色界面降低使用负担,提升专注度
- 在同 WiFi 下用 WebSocket 让多台设备的数据自动同步

### 4.1 功能架构

```

├── 任务管理
│   ├── 任务卡片(创建/编辑/删除/软删除)
│   ├── 任务类型(Idea / Issue / Exploration)
│   ├── 任务状态(待办 / 进行中 / 已完成)
│   └── 任务详情(只读视图)
├── 看板视图
│   ├── 三栏看板(按状态分列)
│   ├── 卡片拖拽(状态流转)
│   ├── 双击列头新建
│   └── 筛选 / 排序 / 搜索
├── 统计分析
│   ├── 核心指标卡片(7 个)
│   ├── 图表(类型环形图 + 趋势折线图)
│   └── 活动热力图(纯 SVG,GitHub 风格)
```

### 4.2 任务管理

#### 4.2.1 任务卡片字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识(UUID v4,旧 Safari 降级) |
| 标题 (title) | string | 是 | 简短描述,≤50 字 |
| 描述 (description) | text | 否 | 详细说明,≤2000 字 |
| 类型 (type) | enum | 是 | `idea` / `issue` / `exploration` |
| 状态 (status) | enum | 是 | `todo` / `in_progress` / `done`,默认 `todo` |
| 优先级 (priority) | enum | 否 | `low` / `medium` / `high`,默认 `medium` |
| 标签 (tags) | string[] | 否 | 自定义标签,每项 ≤20 字符,自动去重 |
| 创建时间 (createdAt) | datetime | 是 | 自动生成(ISO 8601) |
| 更新时间 (updatedAt) | datetime | 是 | 自动刷新(ISO 8601) |
| 完成时间 (completedAt) | datetime \| null | 否 | 状态变更为 `done` 时记录;回退时清空 |
| 删除时间 (deletedAt) | datetime \| null | 否 | 软删除墓碑,用于跨设备删除传播;非空时 UI 隐藏 |

#### 4.2.2 任务类型说明

| 类型 | 标识 | 含义 | 示例 |
|------|------|------|------|
| Idea | idea | 想法、灵感 | 「学一门乐器」 |
| Issue | issue | 疑问、待解决的问题 | 「是否值得深入学习 Rust?」 |
| Exploration | exploration | 探索、调研类任务 | 「调研 2026 年主流 AI Agent 框架」 |

#### 4.2.3 任务状态流转

```
┌──────────┐  开始  ┌──────────────┐  完成  ┌──────────┐
│  待办     │ ────> │   进行中      │ ────> │  已完成    │
│  todo     │       │  in_progress │       │   done   │
└──────────┘       └──────────────┘       └──────────┘
     ↑                    │  暂停               │
     └────────────────────┘                     │
     ↑                                         │
     └───────────── 重新打开 ────────────────────┘
```

- 允许任意方向回退(全部 9 种状态对都合法,含自转)
- 状态变更时自动刷新 `updatedAt`
- 变更为 `done` 时自动记录 `completedAt`;从 `done` 转出时清空

#### 4.2.4 卡片操作

| 操作 | 说明 |
|------|------|
| 新建 | 点击「新建」按钮 / 双击列头空白处,弹出表单填写任务信息 |
| 编辑 | 点击卡片或详情页中的「编辑」入口,修改字段 |
| 删除 | 卡片支持删除,**软删除**(墓碑),UI 立即隐藏,30 天后 GC |
| 状态变更 | 拖拽卡片至目标列,或详情页内切换状态 |
| 详情查看 | 点击卡片进入只读详情页(描述、创建/完成/更新时间) |

#### 4.2.5 软删除与墓碑

- 删除时设置 `deletedAt = now()` 而非物理删除
- UI 与统计层通过 `selectActiveTasks` 过滤 `deletedAt === null` 的任务
- 局域网同步层通过墓碑字段识别「删除事件」并跨设备传播
- `purgeTombstones(ttlDays=30)` 定期 GC 超过 TTL 的墓碑(默认在应用启动时跑一次)

### 4.3 看板视图

#### 4.3.1 布局

- 桌面端:三列并排(待办 / 进行中 / 已完成)
- 移动端:**竖排堆叠**,每列最小高度 280px,可独立滚动
- 每列顶部显示列标题、状态指示点与任务计数
- 卡片以列表形式纵向排列于各列中

#### 4.3.2 卡片展示信息

- 标题(超长省略号截断)
- 类型标识(左侧色条 + 徽章)
- 优先级徽章
- 标签徽章
- 创建/完成时间(相对时间,如「3 天前」)
- 完整 ID(短哈希,便于定位)

#### 4.3.3 筛选 / 排序 / 搜索

| 维度 | 选项 |
|------|------|
| 按类型筛选 | 全部 / idea / issue / exploration |
| 按优先级筛选 | 全部 / high / medium / low |
| 按标签筛选 | 全部 / 指定标签 |
| 排序 | 创建时间 / 优先级 / 更新时间(asc / desc) |
| 搜索 | 全文匹配标题 / 描述 / 标签(case-insensitive) |

#### 4.3.4 拖拽交互

- @dnd-kit 鼠标 / 触控拖拽
- 拖拽时卡片半透明,目标列高亮边框
- 释放后状态立即更新,完成时间按规则联动

#### 4.3.5 双击列头新建

在「待办 / 进行中 / 已完成」标题行空白处双击,直接弹出对应状态的新建表单,省去再次选择状态的步骤。

### 4.4 统计分析

#### 4.4.1 核心指标卡片(7 个)

| 指标 | 说明 |
|------|------|
| 任务总数 | 当前 active 任务数(过滤 `deletedAt`) |
| 待办数 | 状态为 `todo` |
| 进行中数 | 状态为 `in_progress` |
| 已完成数 | 状态为 `done` |
| 完成率 | `done / total` |
| 本周新增 | 近 7 天 `createdAt` |
| 本周完成 | 近 7 天 `completedAt` |

(实现中另有「平均完成周期」指标,作为扩展指标保留。)

#### 4.4.2 图表

| 图表 | 类型 | 实现 | 说明 |
|------|------|------|------|
| 任务类型分布 | 环形图 | ECharts | idea / issue / exploration 占比 |
| 完成与新建趋势 | 双线折线图 | ECharts | 近 30 天每日完成 + 新建数量 |
| 活动热力图 | 53 周 × 7 天网格 | 纯 SVG | GitHub 风格;HELM teal 主色 6 级渐变 |

#### 4.4.3 时间范围

- 统计支持时间范围切换:今日 / 本周 / 本月 / 全部
- 时间范围切换实时重算指标与图表

### 4.5 局域网同步(可选增量)

> 本期为可选能力,默认在开发态启用(`vite-plugin-sync.ts`),生产态需自行托管。

#### 4.5.1 拓扑

```
设备 A ──┐
         ├──► WebSocket Hub (内存) ◄── 设备 B
设备 C ──┘
```

- Hub 在 Vite dev server 内嵌(`/__helm-sync` 端点)
- 每台设备连接后立即推送本地 state;之后每次本地变更都广播
- 新设备连入时,Hub 推送最近一份 snapshot

#### 4.5.2 冲突合并

- **LWW**(Last-Write-Wins):按 `updatedAt` 时间戳比较,新者胜
- **软删除墓碑**:删除通过 `deletedAt` 传播,而非物理移除;接收端识别到墓碑后立即在本地反映删除

#### 4.5.3 已知限制

- Hub 仅内存持久化:`npm run dev` 重启会丢失最新 snapshot,需从某台设备重连重建
- 离线新增 / 删除场景:依赖 tombstone 广播删除;离线新增的内容在重连时通过 LWW 推送
- 无鉴权:任何同 WiFi 用户可加入同步,仅适合家用或受信任网络

### 4.6 移动端适配(iPad / 手机)

| 维度 | 实现 |
|------|------|
| viewport | 响应式 meta;`viewport-fit=cover`;`maximum-scale=5.0` 保留缩放 |
| TopBar | 紧凑布局:小屏仅图标、中屏 icon + 短标签、大屏完整 |
| Kanban | 竖排堆叠,列最小高度 280px,新建按钮仅图标 |
| Modal | 底部弹出 + 圆角;sticky 操作栏;`dvh` 视口防止键盘遮挡 |
| 键盘安全 | 输入框字号 ≥ 16px 防止 iOS 自动缩放;`enterKeyHint="done"`;回车提交 |
| 主题色彩 | 与桌面端完全一致,无差异化色板 |
| 状态徽章 | SyncIndicator 在 sm+ 显示,移动端可关闭网络仅本地使用 |

> iPad Safari < 15.4 兼容性重点:`crypto.randomUUID` 不支持,已用 `getRandomValues` 降级。

## 五、页面设计

### 5.1 设计风格

- **简约**: 去除冗余装饰,信息密度适中,留白合理
- **极客**: 深色主题为主,等宽字体点缀,数据驱动感
- **高效**: 关键操作触手可及,减少点击层级

### 5.2 视觉规范

| 规范项 | 说明 |
|--------|------|

| 同步状态色 | 已连接 = 绿 / 连接中 = 黄 / 出错 = 红 |

## 六、数据模型

### 6.1 任务 (Task)

```typescript
interface Task {
  id: string
  title: string
  description: string
  type: 'idea' | 'issue' | 'exploration'
  status: 'todo' | 'in_progress' | 'done'
  priority: 'low' | 'medium' | 'high'
  tags: string[]
  createdAt: string   // ISO 8601
  updatedAt: string   // ISO 8601
  completedAt: string | null
  deletedAt: string | null   // 软删除墓碑
}
```

### 6.2 存储

- 主存储:IndexedDB(`helm-db` / `tasks` store),首次启动写入 5 条种子
- 辅助存储:localStorage(`helm.ui-preferences`),存筛选 / 排序 / 时间范围 / 指标区显隐
- 数据导入 / 导出:JSON,格式 `{ version: 1, exportedAt, tasks }`
