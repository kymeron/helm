# Quickstart: HELM Personal Dashboard

**Feature**: 001-helm-dashboard
**Date**: 2026-06-19(初版) / 2026-06-20(对齐 M5/M6)
**Status**: 全部场景已可验证

本文档是 HELM 功能的端到端验证指南,用于在实现完成后证明功能可用。不含完整实现代码(见 [tasks.md](tasks.md))。

---

## 前置条件

- Node.js 18+ 已安装
- 现代浏览器(Chrome / Edge / Safari 最新)
- **桌面**: 屏幕宽度 ≥ 1280px
- **扩展** 移动端: iPad Safari 或 iOS Safari(支持 15.4 以下)

## 初始化

```bash
# 安装依赖
npm install

# 启动开发服务器(绑定 0.0.0.0,允许局域网访问)
npm run dev
```

浏览器打开 Vite 输出的本地地址(默认 `http://localhost:5173`)。

> 多设备同步:在同一 WiFi 下,其他设备通过 `http://<电脑 IP>:5173` 访问,自动加入同一 LAN 同步 Hub。

## 验证场景

### 场景 1: 新建任务并出现在看板(US1 / FR-001, FR-002, FR-019)

1. 打开应用,看板三列为空,显示空状态提示。
2. 点击顶部「新建」按钮,弹出模态框。
3. 填写标题「学一门乐器」,类型选 `idea`,优先级选 `high`,标签输入「兴趣」。
4. 提交。
5. **预期**: 「待办」列出现一张卡片,标题为「学一门乐器」,类型徽章为琥珀色(idea),优先级徽章为 high,显示创建时间,列计数为 1。

### 场景 2: 双击列头新建(US1 / FR-019)

1. 在场景 1 基础上,鼠标移到「进行中」列标题行空白处,**双击**。
2. **预期**: 直接弹出新建模态框,状态预填为「进行中」(`defaultStatus = 'in_progress'`)。

### 场景 3: 拖拽状态流转(US1 / FR-003, FR-004)

1. 在场景 1 基础上,将「学一门乐器」卡片从「待办」拖至「进行中」。
2. **预期**: 卡片移至「进行中」列,两列计数更新(待办=0,进行中=1),无完成时间。
3. 将该卡片拖至「已完成」。
4. **预期**: 卡片移至「已完成」列,卡片上显示完成时间,计数更新。

### 场景 4: 状态回退清除完成时间(US1 验收 6 / FR-004)

1. 在场景 3 基础上,将「已完成」的卡片拖回「进行中」。
2. **预期**: 卡片回到「进行中」列,完成时间被清除(卡片不再显示完成时间)。

### 场景 5: 编辑与软删除(US1 / FR-005, FR-006)

1. 点击任一卡片的编辑入口,修改标题为「学吉他」,保存。
2. **预期**: 卡片标题更新为「学吉他」,更新时间刷新。
3. 在任务详情页(点击卡片打开)中点击「删除」按钮。
4. **预期**: 卡片从看板立即消失;但 IndexedDB 中保留软删除墓碑(`deletedAt` 字段被设值);统计区指标同步减少。

### 场景 6: 本地持久化(US1 / FR-007)

1. 新建若干任务(覆盖三种类型与状态)。
2. 刷新浏览器或关闭后重开。
3. **预期**: 所有任务卡片、状态、顺序完整恢复,无丢失。

### 场景 7: 统计指标与图表(US2 / FR-008, FR-009, FR-011)

1. 在已有多种类型与状态任务的前提下,查看页面顶部指标区。
2. **预期**: 显示 7 个指标(总数 / 待办 / 进行中 / 已完成 / 完成率 / 本周新增 / 本周完成),数值与实际任务一致。
3. 查看「类型分布」环形图。
4. **预期**: 三色扇区对应 idea / issue / exploration,hover 显示数量。
5. 查看「完成与新建趋势」双线折线图。
6. **预期**: 近 30 天每日完成 + 新建数量折线,hover 显示某日详情。
7. 查看「活动热力图」。
8. **预期**: 53 周 × 7 天网格,GitHub 风格 6 级渐变,hover 显示具体日期与计数。
9. 新增一条任务或变更一条状态。
10. **预期**: 指标与图表实时刷新,无需手动操作。

### 场景 8: 时间范围切换(US2 / FR-010)

1. 切换时间范围为「本周」。
2. **预期**: 指标与图表按近 7 天数据重算刷新。
3. 切换为「全部」。
4. **预期**: 恢复全量数据视图。

### 场景 9: 筛选 / 排序 / 搜索(US3 / FR-012, FR-013, FR-014, FR-015)

1. 选择类型筛选为 `idea`。
2. **预期**: 三列仅显示 idea 类型卡片,计数更新为筛选后数量。
3. 选择按优先级降序排序。
4. **预期**: 每列内卡片按 high→medium→low 排列。
5. 在搜索框输入关键词「学」。
6. **预期**: 仅显示标题 / 描述 / 标签含「学」的卡片。
7. 点击「清除筛选」。
8. **预期**: 看板恢复全部卡片。

### 场景 10: 数据导出(FR-017)

1. 点击 TopBar 「导出」按钮。
2. **预期**: 浏览器下载 `helm-export-YYYYMMDD-HHMMSS.json`,内容为 `{ version, exportedAt, tasks }` 结构,tasks 含全部任务(含墓碑)。

### 场景 11: 深色主题(FR-018)

1. 打开应用。
2. **预期**: 默认深色主题,背景为深色冷色调,类型徽章颜色为 idea=琥珀 / issue=红 / exploration=青。

### 场景 12: 移动端 — iPad / 手机响应式(US4 / FR-100, FR-101, FR-102)

> **扩展场景**: v1.1 增量。在桌面浏览器打开开发者工具的「设备模式」切换为 iPad / iPhone 也可验证。

1. 在 iPad Safari 上打开 `http://<电脑 IP>:5173`。
2. **预期**: 看板**竖排堆叠**,TopBar 紧凑布局,无横向滚动条;图标与按钮在小屏只显示图标。
3. 点击「新建」按钮。
4. **预期**: 模态框从屏幕底部弹出,圆角对齐顶部。
5. 点击标题输入框,开始输入。
6. **预期**: 输入框字号 ≥ 16px,软键盘弹出后,**提交按钮不被遮挡**(sticky 操作栏在底部可见)。
7. 按下键盘右下角「完成」键。
8. **预期**: 表单自动提交,新任务出现在「待办」列。
9. 选中标题文字开始输入中文(测试 IME)。
10. **预期**: IME 组合态不触发误提交,提交仅在完全字符确认后生效。

### 场景 13: 旧 iPad Safari 兼容(FR-020)

> **扩展场景**: iPad Safari 15.4 以下版本会自动测试降级。

1. 在旧 iPad Safari 上打开 HELM。
2. **预期**: 应用正常加载,**无** `crypto.randomUUID is not a function` 报错;任务可正常创建。
3. 在 DevTools 控制台验证:
   - `crypto.randomUUID` 不存在时,内部调用 `getRandomValues` 走 RFC 4122 v4 路径。

---

### 场景 14: 局域网同步 — 双设备(US5 / FR-200, FR-201, FR-202)

> **扩展场景**: v1.1 增量。需要两台设备在同一 WiFi 下。

1. 设备 A 打开 `http://<电脑 IP>:5173`,设备 B 同样打开同一地址。
2. 设备 A 创建一条新任务「跨设备测试」。
3. **预期**: 设备 B 在 1 秒内看到该任务(由 LWW 同步),且 B 的 IndexedDB 也已写入。
4. 设备 A 编辑该任务标题为「跨设备测试 - 已编辑」。
5. **预期**: 设备 B 上对应任务标题同步更新(LWW 按 `updatedAt` 取新者)。
6. 设备 A 删除该任务(详情页「删除」按钮)。
7. **预期**: 设备 B 上对应任务消失(通过 `deletedAt` 墓碑传播,接收端在 1 秒内反映)。

### 场景 15: 局域网同步 — 离线与重连(US5 / FR-203)

1. 设备 A 在线,设备 B 关闭 WiFi。
2. 设备 B 继续操作: 新建任务「离线任务」、编辑某任务、删除另一任务。
3. **预期**: 设备 B 仍能正常使用(纯本地模式);数据写入 B 的 IndexedDB。
4. 设备 B 重新连接 WiFi。
5. **预期**: B 收到 Hub 的最新 snapshot 并与之对齐;LWW 合并保证最终一致。

### 场景 16: 局域网同步 — Hub 重启(US5)

1. 在 PC 上停止 `npm run dev`(`Ctrl+C`)再重启。
2. 设备 A 重新连接。
3. **预期**: A 推送本地最新 state 作为新的 Hub snapshot;其他设备重连后能收到。
4. TopBar 同步徽章会从「已同步」→「重连中」→「已同步」切换。

### 场景 17: 同步状态徽章(FR-204)

1. 关闭电脑 WiFi,在浏览器打开 HELM。
2. **预期**: TopBar 同步徽章显示「未连接」灰色圆点。
3. 重新连接 WiFi。
4. **预期**: 徽章先显示「连接中」黄色脉动 →「已同步」绿色。
5. hover 徽章。
6. **预期**: 显示推送次数 / 接收次数(便于调试)。

全部 17 个场景通过即视为 HELM 功能端到端可用,满足 [spec.md](spec.md) 的 US1–US5 与 FR-001~FR-021、FR-100~FR-204。

## 已知限制(非测试失败)

- Hub 仅内存,dev server 重启丢失 snapshot → 场景 16 已说明预期
- 离线新增可能在重连时丢失(若对端恰好删除了同一时间窗口的不相关任务) → 场景 15 边缘情况,LWW 固有限制
- 无鉴权,公开网络应自托管 → 文档已说明

## 跨平台访问入口

| 设备 | URL | 备注 |
|------|-----|------|
| 本机 | `http://localhost:5173` | 开发默认 |
