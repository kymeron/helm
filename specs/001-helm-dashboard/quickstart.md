# Quickstart: HELM Personal Dashboard

**Feature**: 001-helm-dashboard
**Date**: 2026-06-19

本文件是 HELM 功能的端到端验证指南，用于在实现完成后证明功能可用。不含完整实现代码（见 tasks.md）。

---

## 前置条件

- Node.js 18+ 已安装
- 现代浏览器（Chrome/Edge/Safari 最新版）
- 屏幕宽度 ≥ 1280px

## 初始化

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

浏览器打开 Vite 输出的本地地址（默认 http://localhost:5173）。

## 验证场景

### 场景 1：新建任务并出现在看板（US1 / FR-001, FR-002）

1. 打开应用，看板三列为空，显示空状态提示。
2. 点击顶部"新建"按钮，弹出模态框。
3. 填写标题"学一门乐器"，类型选 `idea`，优先级选 `high`，标签输入"兴趣"。
4. 提交。
5. **预期**: "待办"列出现一张卡片，标题为"学一门乐器"，类型徽章为琥珀色（idea），优先级徽章为 high，显示创建时间，列计数为 1。

### 场景 2：拖拽状态流转（US1 / FR-003, FR-004）

1. 在场景 1 基础上，将"学一门乐器"卡片从"待办"拖至"进行中"。
2. **预期**: 卡片移至"进行中"列，两列计数更新（待办=0，进行中=1），无完成时间。
3. 将该卡片拖至"已完成"。
4. **预期**: 卡片移至"已完成"列，卡片上显示完成时间，计数更新。

### 场景 3：状态回退清除完成时间（US1 验收 6 / FR-004）

1. 在场景 2 基础上，将"已完成"的卡片拖回"进行中"。
2. **预期**: 卡片回到"进行中"列，完成时间被清除（卡片不再显示完成时间）。

### 场景 4：编辑与删除（US1 / FR-005, FR-006）

1. 点击任一卡片的编辑入口，修改标题为"学吉他"，保存。
2. **预期**: 卡片标题更新为"学吉他"，更新时间刷新。
3. 点击卡片删除入口，弹出二次确认。
4. 确认删除。
5. **预期**: 卡片从看板移除，列计数减 1。

### 场景 5：本地持久化（US1 / FR-007）

1. 新建若干任务（覆盖三种类型与状态）。
2. 刷新浏览器或关闭后重开。
3. **预期**: 所有任务卡片、状态、顺序完整恢复，无丢失。

### 场景 6：统计指标与图表（US2 / FR-008, FR-009, FR-011）

1. 在已有多种类型与状态任务的前提下，查看页面顶部指标区。
2. **预期**: 显示 7 个指标（总数/待办/进行中/已完成/完成率/本周新增/本周完成），数值与实际任务一致。
3. 查看"类型分布"环形图。
4. **预期**: 三色扇区对应 idea/issue/exploration，hover 显示数量。
5. 查看"完成趋势"折线图。
6. **预期**: 近 30 天每日完成数量折线，hover 显示某日详情。
7. 新增一条任务或变更一条状态。
8. **预期**: 指标与图表实时刷新，无需手动操作。

### 场景 7：时间范围切换（US2 / FR-010）

1. 切换时间范围为"本周"。
2. **预期**: 指标与图表按近 7 天数据重算刷新。
3. 切换为"全部"。
4. **预期**: 恢复全量数据视图。

### 场景 8：筛选与排序（US3 / FR-012, FR-013, FR-014）

1. 选择类型筛选为 `idea`。
2. **预期**: 三列仅显示 idea 类型卡片，计数更新为筛选后数量。
3. 选择按优先级降序排序。
4. **预期**: 每列内卡片按 high→medium→low 排列。
5. 点击"清除筛选"。
6. **预期**: 看板恢复全部卡片。

### 场景 9：数据导出（FR-016）

1. 点击顶部"导出"按钮。
2. **预期**: 浏览器下载 `helm-export-YYYYMMDD-HHMMSS.json`，内容为 `{ version, exportedAt, tasks }` 结构，tasks 含全部任务。

### 场景 10：深色主题（FR-017）

1. 打开应用。
2. **预期**: 默认深色主题，背景为深色冷色调，类型徽章颜色为 idea=琥珀/issue=红/exploration=青。
