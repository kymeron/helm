# Contract: HELM Cloud Sync API

**Feature**: [spec.md](../spec.md)
**Base Path**: `/api/sync`

## Overview

HTTP-based snapshot sync API for HELM deployed on Vercel. All endpoints are stateless serverless functions backed by Vercel KV (Redis). The client owns the task schema; the server only stores and retrieves an opaque JSON snapshot per user token.

## Authentication

No traditional authentication. A URL-safe `token` identifies a user's data partition.

- Token is provided via query parameter `?token=xxx` on every request.
- Token is generated by the client on first visit and persisted in `localStorage`.
- Sharing the URL with `?token=xxx` grants access to the same dataset.

## Endpoints

### GET /api/sync?token={token}

Fetch the latest server snapshot for the given token.

**Request**:

```http
GET /api/sync?token=a1b2c3d4e5f6g7h8 HTTP/1.1
```

**Success Response** (`200 OK`):

```json
{
  "tasks": [
    {
      "id": "task-001",
      "title": "示例任务",
      "description": "",
      "status": "todo",
      "priority": "medium",
      "tags": ["idea"],
      "createdAt": "2026-06-21T10:00:00.000Z",
      "updatedAt": "2026-06-21T10:00:00.000Z",
      "completedAt": null,
      "deletedAt": null
    }
  ],
  "updatedAt": "2026-06-21T10:00:00.000Z"
}
```

**Empty Snapshot Response** (`200 OK`):

```json
{
  "tasks": [],
  "updatedAt": null
}
```

**Error Responses**:

- `400 Bad Request`: Missing `token` query parameter.
- `500 Internal Server Error`: KV read failure or malformed stored data.

### POST /api/sync?token={token}

Push the client's current task snapshot to the server.

**Request**:

```http
POST /api/sync?token=a1b2c3d4e5f6g7h8 HTTP/1.1
Content-Type: application/json

{
  "tasks": [
    {
      "id": "task-001",
      "title": "示例任务",
      "description": "",
      "status": "todo",
      "priority": "medium",
      "tags": ["idea"],
      "createdAt": "2026-06-21T10:00:00.000Z",
      "updatedAt": "2026-06-21T10:05:00.000Z",
      "completedAt": null,
      "deletedAt": null
    }
  ]
}
```

**Success Response** (`200 OK`):

```json
{
  "ok": true,
  "updatedAt": "2026-06-21T10:05:00.000Z"
}
```

**Error Responses**:

- `400 Bad Request`: Missing `token` or `tasks` is not an array.
- `413 Payload Too Large`: Snapshot exceeds maximum allowed size (default 1 MB).
- `500 Internal Server Error`: KV write failure.

## Error Shape

All errors use the following JSON body:

```json
{
  "error": "human-readable message"
}
```

## Constraints

- Server does not interpret task schema beyond JSON validation and size limits.
- Server does not resolve conflicts; merging is performed client-side.
- Each write overwrites the previous snapshot for the token.
- KV TTL is refreshed on every successful write.
